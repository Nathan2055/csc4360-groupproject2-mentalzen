1

Project Structure

  • Each group will select one project topic from the provided list.
  • Every project includes a clear set of required features and extended functionalities.
  • Development will follow two main milestones:
    • Milestone 1: Focus on UI design, layout, and basic navigation.
    • Milestone 2: Implement full functionality, database/local storage integration, and testing.
2

AI Usage Guidelines

  • Students may use AI tools (ChatGPT, GitHub Copilot, Gemini, etc.) as assistants for learning, debugging, and brainstorming.
  • AI should not be used to generate entire codebases or bypass original development work.
  • Each team must maintain an AI Transparency Log (e.g., AI_Usage_Log.md in GitHub) that records:
    • Date and AI tool used
    • What was asked or generated
    • How it was applied
    • Reflection on what was learned

Note:

We will be closely monitoring GitHub commits during grading to ensure progress is natural and consistent. Any irregularities or lack of updates will affect the grade.

3

Version Control & Progress

  • The use of GitHub is mandatory for project submission and development.
  • Commits should clearly show gradual development.
  • Teams must collaborate actively, with commits reflecting each member's contribution.
  • Lack of version control activity may result in grade deductions.
4

Milestones Submission

  • Each milestone will be reviewed via GitHub commits and brief progress summaries.
  • Make specific Git commits after reaching each milestone (e.g., "Milestone 1 achieved").
  • Late submissions or incomplete milestones will result in point deductions.
5

Submission Requirements

  • Final project submission must include the following:
    • GitHub Repository URL (mandatory for all teams)
    • Team YouTube Demo Video Link
    • PowerPoint Slide Deck
    • APK File for Testing
6

Bonus Credit

  • Groups that add extra or innovative functionalities beyond the project description may receive bonus credit for creativity and technical effort.
